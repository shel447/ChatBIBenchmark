# 评测任务弹窗与进度条设计

## 背景
用户反馈“新增评测任务”按钮无响应，并希望在任务列表与任务详情显示进度条效果。本次改造需提供弹窗表单配置评测任务，并与后端 `/api/runs` 真实交互。

## 目标
- “新增评测”按钮打开弹窗表单，完成任务配置并调用后端创建任务。
- 任务列表与任务详情显示进度条（基于已执行/总用例）。
- 失败时保留表单输入并给出明确错误提示。

## 非目标
- 不实现真实调度执行引擎或进度推进逻辑。
- 不引入复杂表单校验规则（仅必填）。
- 不重构为完整前端框架，保持原生 HTML/CSS/JS。

## 交互方案
### 弹窗
- 触发：点击“新增评测”按钮。
- 关闭方式：点击遮罩、点击右上角关闭按钮、点击“取消”。
- 表单字段（必填）：
  - 任务名称
  - 用例集
  - 环境
  - 指标参数集
  - 用例集执行次数
- 主按钮：`创建并启动`，提交期间显示 loading（禁用）。
- 错误提示：表单底部红色提示条。

### 进度条
- 任务列表“进展”列：文本 `已执行/总数` + 细条进度条。
- 任务详情“进展”卡片：显示同样进度条。
- 计算规则：
  - 总数为 0 时显示 0%。
  - 进度条宽度 = `executed / total * 100%`。

## 数据流
1. 点击“新增评测” → 打开弹窗。
2. 表单校验通过 → `POST /api/runs`。
3. 成功：关闭弹窗，插入任务列表首行（进展 0/0，准确率 0%）。
4. 失败：保留输入，显示错误提示。

## 错误处理
- 必填缺失：提示“请填写所有必填项”。
- 接口失败：提示“创建失败，请稍后重试”。

## 组件与样式
- `modal` 容器 + `modal-backdrop` 遮罩。
- `modal-card`：与现有 `panel` 风格一致，阴影与圆角匹配。
- `progress-bar`：浅灰底 + 主题色填充。
- 复用现有字体与颜色变量，保证风格统一。
