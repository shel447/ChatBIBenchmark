# UI 调整与管理界面扩展设计

## 背景
现有前端为静态单页展示，用户需要将操作入口下沉到对应页面，并补齐评测任务的关键要素展示，同时新增环境配置与评测指标管理页面。

## 目标
- “导入用例”仅出现在“用例集”页面，“新增评测”仅出现在“任务列表”页面。
- 评测任务列表展示环境、用例集、启动/结束时间与指标参数集，支持启动与结束时间各自排序按钮（静态）。
- 新增一级菜单：环境配置、指标管理，并提供对应列表页。
- 保持中文文案与当前视觉风格一致。

## 信息架构
- 一级菜单并列：总览、任务列表、用例集、环境配置、指标管理。
- 详情页面仍通过列表跳转进入（现有交互保持）。

## 页面设计
### 任务列表（评测任务）
- 顶部控制区：用例集过滤下拉（静态）。
- 表头：为“启动时间”“结束时间”各提供排序按钮（上下），默认启动时间倒排高亮。
- 列新增：用例集、环境、启动时间、结束时间、指标参数集。
- 右侧操作：新增评测按钮。

### 用例集
- 右侧操作：导入用例按钮。

### 环境配置
- 列表字段：环境名、基础地址、状态、最近更新。
- 操作按钮：新增环境、编辑（静态占位）。

### 指标管理
- 列表字段：参数集名称、描述、覆盖指标/阈值、最近更新。
- 操作按钮：新增参数组合、编辑（静态占位）。

## 交互与范围
- 所有排序与过滤仅为静态 UI，不接入真实数据。
- 不改动后端接口与数据模型。
